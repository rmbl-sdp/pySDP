# pySDP

Native Python client for the RMBL Spatial Data Platform. Cloud-optimized GeoTIFFs on S3 (`rmbl-sdp.s3.us-east-2.amazonaws.com`), accessed lazily via GDAL VSICURL and `rioxarray`/`xarray`.

## Layout

- `src/pysdp/__init__.py` — re-exports the public API (`get_catalog`, `get_metadata`, `open_raster`, `open_stack`, `extract_points`, `extract_polygons`, `download`) + constants + `__version__`
- `src/pysdp/catalog.py` — `get_catalog()`, `get_metadata()` (Phase 1)
- `src/pysdp/raster.py` — `open_raster()`, `open_stack()` (Phase 3)
- `src/pysdp/extract.py` — `extract_points()`, `extract_polygons()` (Phase 4)
- `src/pysdp/download.py` — `download()` (Phase 5)
- `src/pysdp/stac.py` — pystac / pystac-client wrapper (Phase 1)
- `src/pysdp/constants.py` — `SDP_CRS` (EPSG:32613), `DOMAINS`, `TYPES`, `RELEASES`, `TIMESERIES_TYPES`, `VSICURL_PREFIX`, `CATALOG_ID_NCHAR`. Mirrors rSDP's `R/constants.R`.
- `src/pysdp/_resolve.py` — time-slice resolvers per TimeSeriesType. Behavior-preserving port of rSDP's `R/internal_resolve.R` (Phase 2).
- `src/pysdp/_validate.py` — argument validation, port of rSDP's `R/internal_validate.R` (Phase 2).
- `src/pysdp/_catalog_data.py` — packaged CSV loader + staleness warning (Phase 1).
- `src/pysdp/io/vsicurl.py` — GDAL VSICURL helpers + cloud-tuned defaults (Phase 3 core, Phase 7 Hub tuning).
- `src/pysdp/io/template.py` — `{year}/{month}/{day}` URL templating (Phase 3).
- `src/pysdp/data/` — packaged catalog CSV snapshot (Phase 1).
- `tests/test_smoke.py` — Phase 0 shape tests; more tests land alongside each phase.
- `SPEC.md` — v0.1 specification and phased plan.
- `ROADMAP.md` — post-v0.1 Hub/Dask integrations.

## Conventions

- **Import name = dist name = `pysdp`.** (The shorter `sdp` was squatted on PyPI.)
- **src-layout**, `hatchling` build backend, `hatch-vcs` for version-from-git-tag.
- **Type-annotated public API**; `from __future__ import annotations` everywhere so heavy deps (xarray, geopandas) are `TYPE_CHECKING`-only where sensible.
- **Ruff** for lint + format; **mypy strict** on `src/pysdp/`; **pytest** for tests.
- **PEP 735 dependency groups** (`[dependency-groups]`) for dev-only deps, not `[project.optional-dependencies]`.
- **Optional extras**: `[stac]`, `[exact]`, `[download]`, `[viz]`, `[dask]`, `[hub]`, `[all]`. `[hub]` self-references `pysdp[stac]` so `odc-stac` isn't duplicated.
- All SDP rasters are `EPSG:32613` (UTM 13N); hard-coded in `constants.py`.
- Catalog IDs are 6-char codes like `R3D009`, `BM012` (Release/Basemap + type + number).
- `TimeSeriesType` ∈ {`Single`, `Yearly`, `Seasonal`, `Monthly`, `Daily`} drives the resolver dispatch.
- URL templates use `{year}`, `{month}`, `{day}` placeholders.
- **Return types**: raster access returns `xarray.Dataset` with one variable per product (variable name = COG filename minus `.tif`). Extraction returns `geopandas.GeoDataFrame`.
- **Time coords** are uniformly `pd.DatetimeIndex`: Daily = actual date, Monthly = first-of-month, Yearly = Jan 1.
- **Behavior carry-overs from rSDP** (do NOT change during port): daily default clip to 30 layers; anchor-day `seq(by="month"/"year")` semantics for Yearly/Monthly date ranges; error-on-empty-overlap; URL-branch skips scale/offset; catalog CSV is a packaged snapshot with 6-month staleness warning (`SDP_STALENESS_MONTHS` env).

## Common commands

```bash
# Dev environment (preferred: uv)
uv sync --all-groups --all-extras

# Or classic pip
pip install -e ".[all]"

# Lint, format, type-check, test
uv run ruff check .
uv run ruff format .
uv run mypy src/pysdp
uv run pytest -m "not network"

# Integration tests (hit live S3)
uv run pytest -m network
```

## Things to be careful about

- **Live tests are network-gated** via `@pytest.mark.network`. Skipped in default CI. A failing network test could mean a bug *or* a moved/renamed SDP dataset — same caveat as rSDP.
- **Catalog drift**: `src/pysdp/data/SDP_product_table_*.csv` is a frozen snapshot. Canonical source is the CSV on S3; regenerate via `scripts/update_catalog.py`. `get_catalog(source="live")` always hits fresh.
- **Scale/offset** on rasters applied as CF `scale_factor` / `add_offset` attrs — xarray honors them on `decode_cf()`.
- **Don't edit `src/pysdp/_version.py`** — generated by `hatch-vcs` at build time; git-ignored.
- **Don't mutate global `os.environ`** for GDAL config — scope via `rasterio.Env(...)` or `os.environ.setdefault(...)` so user overrides survive.
- **Anchor-day semantics** in `_resolve.py` Yearly/Monthly date-range branches are load-bearing; preserve them verbatim from rSDP.
